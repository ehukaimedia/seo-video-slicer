Why it ships motion the cheap way

Frames are the product. The model just wires them in.

Generating hero imagery from scratch is slow and non-deterministic. Hand the model a finished package instead.

01 / Token burn

Eliminated.

Ship ready-made frames, not generated imagery. The model copies a folder and writes a few lines — it never spends image tokens drawing motion that already exists.

02 / Core Web Vitals

LCP-safe.

WebP frame sequences are lazy-loadable and CLS-free — a fraction of the weight of shipping video or a heavy JS animation library. Engineered to help your vitals, not hurt them.

03 / Integration

Drop-in.

Copy the folder to public/, open index.html, and it animates now — no server, no build step, zero external requests.

Run it

Two ways in.

Zero-clone with uvx, or clone and launch the full dark studio.

uvx · zero clone 
$ uvx --from <release-wheel-url> seo-video-slicer

Grab the wheel URL from the latest Release. Needs ffmpeg and node on your PATH.

clone · full studio 
$ git clone https://github.com/ehukaimedia/seo-video-slicer \
    && cd seo-video-slicer \
    && ./start.command

Sets up the venv, builds the UI, and serves the slicer on localhost:8000.
