The package is
the product.

A premium, single-purpose video slicer that turns a 5–10s clip from any source into a drop-in WebP animation package — frames, a working self-contained scroll player, a manifest, and an offline verify gate — that a frontier model wires into any repo at near-zero image-generation token burn.

Intent · the inversion

We invert smart-image-animations.

The source repo made animation templates the product and grew a studio around them (GSAP generator, MLX/Gemma chat, a multi-project dashboard). We invert it: the optimized, sliced-image package is the product. We do not generate the animation. The downstream repo’s frontier model already “does the backend” — it just needs premium raw material.

01 / token burn

Eliminate generation cost

Asking a model to create hero imagery from scratch is slow, expensive, and non-deterministic. A tiny WebP frame sequence + a working player means near-zero tokens and enterprise motion on the first try.

02 / SEO & CWV

Engineered for Core Web Vitals

WebP frame sequences are LCP-safe, lazy-loadable, CLS-free, and a fraction of the weight of shipping video or a heavy JS animation library. The package helps CWV, never hurts it.

03 / one-pass integration

Drop in & ship

Copy the folder to public/, open index.html, see premier scroll animation in 10 seconds, then tell the model “use this” — integration in a single pass. Five stars is the bar; anything that reads as a template fails.

The package contract · define this FIRST (§8)

The exported folder, visualized.

Self-contained, zero external requests, relative paths, works offline. The lean §8 contract — not the source’s multi-doc package ceremony. We take the verify.mjs + fingerprint pattern, not the doc pile.

<slug>-animation/ ├── frames/ │ ├── frame_000.webp # zero-padded, sortable │ ├── frame_001.webp │ └── … frame_NNN.webp # 20–80, q82–90 ├── index.html # self-contained player ├── manifest.json # locked vs safe zones ├── verify.mjs # offline gate · zero deps ├── README.md # ≤200 lines · 3 recipes └── PROMPT.md (optional) # drop-in LLM instructions

manifest.json · locked_zones

The integrity of the package. A downstream model must not touch these — do not regenerate the images.

frames/*.webp bytes frame_NNN zero-pad ordering cover-fit single-canvas render reduced-motion fallback

manifest.json · safe_zones

The customization surface. A model is free to adapt these to its host repo without breaking the contract.

accent color headline / overlay copy scroll distance easing container height framework wrapper

index.html — locked player techniques

Distilled to one deterministic player (cherry-picked from the source apple_scroll_zoom template, GSAP-studio metadata stripped):

  • Single <canvas>, cover-fit render (scale = max(cw/iw, ch/ih), centered) — no duplicate blurred background layer.
  • DPR scaling for retina sharpness.
  • Parallel preload with an onerror-tolerant counter — a 404 never freezes playback.
  • Frame index = scroll progress × (frameCount − 1); a no-dependency rAF fallback is preferred for true zero-external-requests (GSAP from cdnjs is the only permitted external request).
  • prefers-reduced-motion: reduce → static hero frame; brand still reads. System fonts only; inline CSS/JS; opens with no server.

Current vs. target, boundaries, interfaces, sequence, gates.

Current vs. target behavior

Current — smart-image-animations studio

  • Studio generates GSAP/ScrollTrigger animations in-app.
  • MLX/Gemma model server + chat Animation Director.
  • Multi-project dashboard / job gallery is a product surface.
  • Erase = cv2.inpaint(INPAINT_TELEA) per frame: smears, no feather, no temporal coherence.
  • Hardcoded if (end - start) > 10.0 duration wall.
  • 4-format frame sprawl checked everywhere; gold-standard multi-doc package ceremony.
  • Dark Void-Black / Electric-Blue brand; Next.js route tree.

Target — this lean slicer

  • No in-app generator. ONE deterministic reference player ships in the package.
  • No local LLM. ffmpeg + OpenCV + optional neural erase only.
  • One session = one working clip; a minimal recents list, not a CRM.
  • Two-tier erase: LaMa/IOPaint premium + enhanced OpenCV INPAINT_NS fallback, both with temporal coherence.
  • MAX_SLICE_SECONDS=60 constant + frame-budget meter as the real governor.
  • JPEG preview → WebP package, one format per stage; lean §8 contract.
  • Impeccable light editorial chrome + neutral-dark media stage; single-page Vite/React.

Data flow

video (POST /upload, thumbnail) → trim (in/out, default 10s, budget-governed) → frames (ffmpeg fps → JPEG preview) → review (filmstrip exclude, finalize) → crop (manual or OpenCV auto + watermark enforcer) → erase (two-tier inpaint, temporal coherence) → webp (q82–90, zero-pad) → rename (SEO prefix) → package (assemble + manifest + verify.mjs gate + zip) → download + share URLs. A single linear path: no studio branch, no model round-trip.

Acceptance tests — verify.mjs gates G1–G7 (§8)

node verify.mjs exits non-zero on any failed gate. The export pipeline runs it before declaring success.

G1Asset closure. Every frame referenced by index.html exists in frames/.
G2Contiguity. Frames are zero-padded, contiguous frame_000…frame_NNN; count matches manifest.frames.count.
G3Self-contained. No http(s)://…/<local-asset> leaks, no scratch/localhost paths; relative ./frames/… only.
G4Technique check. index.html contains cover-fit render, DPR scaling, preload-with-onerror, and a prefers-reduced-motion block.
G5Manifest integrity. manifest.json parses, has schema + fingerprint; fingerprint matches recomputed value.
G6Docs. README.md exists, ≤200 lines, has Iframe / React / Vanilla headings. PROMPT.md optional.
G7Weight budget. Total package within budget; each WebP within per-frame budget (flag oversized). The backstop that keeps any duration honest.

Duration & frame-budget policy (§5.1)

Seconds are configurable. Frame weight is the governor.

Default DEFAULT_SLICE_SECONDS = 10 (the hero sweet spot, auto-applied on import). Ceiling MAX_SLICE_SECONDS = 60 is a single config constant (backend env + frontend mirror) — raising it later is a one-line change; never hardcode 10/60 elsewhere. The real safety rail is the frame-budget meter: it projects live frame count and package weight, and auto-suggests a lower fps as duration rises. verify.mjs G7 is the backstop that keeps any chosen duration honest to the 5-star CWV bar.
